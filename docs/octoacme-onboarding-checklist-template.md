# OctoAcme Team Onboarding Checklist

## Purpose
This checklist ensures new team members are properly onboarded to OctoAcme projects with clear understanding of their role, responsibilities, and how to collaborate effectively with the team.

---

## General Onboarding (All Roles)

### First Day
- [ ] Complete HR onboarding and receive necessary accounts/credentials
- [ ] Set up development environment and tools (GitHub, Slack, project boards, etc.)
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles & Personas](octoacme-roles-and-personas.md) - understand your role and key interactions
- [ ] Introduce yourself to the team during standup or team meeting
- [ ] Schedule 1:1 meetings with key collaborators identified in your role description
- [ ] Add yourself to relevant communication channels and mailing lists

### First Week
- [ ] Review current project documentation in the `docs/` folder
- [ ] Understand the current sprint/iteration goals and timeline
- [ ] Shadow relevant meetings (standups, planning, reviews) to understand team rhythm
- [ ] Review the RACI chart for current project to understand your responsibilities
- [ ] Review the project board to understand current work in progress
- [ ] Complete security and compliance training if applicable
- [ ] Identify and document any questions or clarifications needed

### First Month
- [ ] Participate actively in at least one retrospective
- [ ] Complete your first significant contribution or deliverable
- [ ] Provide feedback on the onboarding process to your Project Manager
- [ ] Build relationships with all team members you'll regularly interact with

## Role-Specific Onboarding

### Product Manager

**Week 1**
- [ ] Review product vision, strategy, and roadmap documents
- [ ] Understand market positioning and competitive landscape
- [ ] Review historical product metrics and success criteria
- [ ] Meet with Product Owner to align on strategic priorities
- [ ] Meet with key stakeholders and understand their expectations

**Week 2-4**
- [ ] Shadow stakeholder meetings and executive reviews
- [ ] Review customer feedback, user research, and analytics
- [ ] Understand product release history and upcoming milestones
- [ ] Participate in roadmap planning discussions

---

### Product Owner

**Week 1**
- [ ] Review current product backlog and prioritization
- [ ] Understand acceptance criteria standards and Definition of Done
- [ ] Review user stories and feature specifications format
- [ ] Meet with Product Manager to understand strategic context
- [ ] Meet with Development Team to understand technical constraints
- [ ] Shadow backlog refinement sessions

**Week 2-4**
- [ ] Lead backlog refinement session with support from outgoing PO or PM
- [ ] Participate in sprint planning and review meetings
- [ ] Write first user stories and acceptance criteria
- [ ] Conduct stakeholder demo and gather feedback

---

### Project Manager

**Week 1**
- [ ] Review project charter, timeline, and current status
- [ ] Understand project risks, dependencies, and constraints
- [ ] Review existing project documentation and templates
- [ ] Meet with all team leads and understand their concerns
- [ ] Access project tracking tools and dashboards
- [ ] Review communication plan and stakeholder list

**Week 2-4**
- [ ] Facilitate first project status meeting
- [ ] Update risk register and dependency tracker
- [ ] Create or update project plan as needed
- [ ] Establish regular cadence of stakeholder updates
- [ ] Document any process improvements identified

---

### Developer

**Week 1**
- [ ] Clone repositories and complete local development setup
- [ ] Run application locally and understand architecture
- [ ] Review coding standards, style guides, and contribution guidelines
- [ ] Review CI/CD pipeline and testing requirements
- [ ] Understand branching strategy and PR workflow
- [ ] Complete first code review (review someone else's PR)
- [ ] Fix first small bug or documentation issue to practice workflow

**Week 2-4**
- [ ] Complete first feature development with PR and tests
- [ ] Participate in technical design discussion
- [ ] Understand deployment process and monitoring tools
- [ ] Learn debugging and troubleshooting procedures

---

### Quality Assurance Lead

**Week 1**
- [ ] Review current test strategy and quality standards
- [ ] Understand test automation framework and tools
- [ ] Review defect tracking process and current open issues
- [ ] Access test environments and testing tools
- [ ] Review Definition of Done and acceptance criteria standards
- [ ] Meet with Developers to understand testing pain points

**Week 2-4**
- [ ] Execute test cases for current sprint features
- [ ] Review and improve test documentation
- [ ] Participate in sprint planning to ensure testability
- [ ] Create or update test plans for upcoming features
- [ ] Identify quality metrics and reporting approach

---

### UX Designer

**Week 1**
- [ ] Review existing design system and UI component library
- [ ] Understand current user personas and user research findings
- [ ] Review accessibility standards and requirements (WCAG)
- [ ] Access design tools (Figma, Sketch, Adobe XD, etc.)
- [ ] Review past usability testing results and feedback
- [ ] Meet with Product Owner to understand feature roadmap

**Week 2-4**
- [ ] Create first design mockups or wireframes
- [ ] Participate in design review session
- [ ] Conduct or observe user research session
- [ ] Collaborate with Developers on design implementation
- [ ] Document design patterns and contribute to design system

---

### Business Analyst

**Week 1**
- [ ] Review existing requirements documentation and specifications
- [ ] Understand business processes and workflows
- [ ] Review current business rules and logic
- [ ] Meet with stakeholders to understand business objectives
- [ ] Access business intelligence tools and reporting systems
- [ ] Review requirements gathering templates and standards

**Week 2-4**
- [ ] Facilitate first requirements gathering session
- [ ] Create process flow diagrams or business rules documentation
- [ ] Validate requirements with stakeholders and technical team
- [ ] Participate in backlog refinement with Product Owner
- [ ] Document first set of functional specifications

---

## Onboarding Checkpoints

### Manager/Lead Check-ins

**End of Week 1**
- Review checklist completion
- Address any blockers or concerns
- Clarify role expectations
- Ensure access to all necessary tools

**End of Week 2**
- Discuss first observations and questions
- Review upcoming responsibilities
- Identify knowledge gaps

**End of Month 1**
- Comprehensive onboarding review
- Gather feedback on onboarding experience
- Set goals for next 30-60 days
- Discuss career development opportunities
